# Розробка застосунків баз даних

## Інженерія програмного забезпечення | Medina County Career Center

## Що ми будуємо

На уроках 01–02 ви завантажували дані з CSV файлів та досліджували їх за допомогою pandas. **Сьогодні ми піднімаємось на новий рівень**: замість CSV файлів ми зберігатимемо дані в реальній **базі даних (database)** — тій же технології, що живить Instagram, Spotify та ваш банк.

Чому? Тому що бази даних:
- **Швидше** знаходять інформацію
- **Безпечніше** захищають дані
- **Розумніше** обробляють складні питання

На кінець уроку у вас буде робоча SQLite база даних і ви знатимете, як ставити їй питання мовою SQL.

## Дані та інформація

**Дані** = сирі факти без контексту
```
17, "Alice", 3.75, True
```

**Інформація** = дані, які організовані та мають значення
```
"Студентка Аліса 17 років зі середнім балом 3.75 і на даний момент навчається."
```

Коли сирі факти отримують контекст, вони стають корисною інформацією. Ваш телефон зберігає мільйони точок даних. Коли ви ставите питання Siri чи Google, вона обробляє дані в інформацію — часто використовуючи SQLite базу даних позаду сцени.

---

## Що таке база даних?

**База даних (Database)** — це організована колекція пов'язаних даних, керована програмним забезпеченням.

Подумайте про це так:
- **CSV файл** = безладна купа паперу з числами
- **База даних** = ці числа акуратно організовані в шафі з правилами про те, що куди іде

```
CSV: numbers.csv
База даних: customers.db (файл на вашому комп'ютері)
```

---

## Чому використовувати базу даних замість CSV?

| Проблема з CSV | Як база даних її вирішує |
|---|---|
| Кожен може вводити неправильні дані | Правила забезпечують коректні дані |
| Одні й ті ж дані повторюються | Зберігайте один раз, користуйтесь всюди |
| Важко ставити складні питання | SQL робить складні запити легкими |
| Тільки одна людина може користуватися одночасно | Безпечний одночасний доступ багатьох користувачів |
| Немає захисту конфіденційних даних | Вбудований контроль доступу |

## Типи баз даних

Існує багато видів баз даних. Ось головні:

| Тип | Як це працює | Приклади |
|---|---|---|
| **Реляційна (Relational)** | Дані в таблицях (рядки й колонки) | SQLite, MySQL, PostgreSQL |
| **NoSQL** | Дані як документи або пари ключ-значення | MongoDB, Firebase |
| **Хмарна (Cloud)** | Розташована в Інтернеті, оплата за використання | AWS RDS, Google BigQuery |

> **Ми використовуємо SQLite** — це безплатно, на основі файлів (один файл `.db`), і не потребує сервера. Ідеально для навчання.

---

## Що таке DBMS?

**DBMS** — це **система управління базою даних (Database Management System)**.

Це програмне забезпечення, яке:
- Безпечно зберігає ваші дані
- Організує їх в таблицях
- Захищає їх правилами доступу
- Витягує їх, коли ви ставите питання

SQLite **це** DBMS. Коли ви створюєте `titanic.db`, ви створюєте SQLite базу даних.

```python
import sqlite3

# Створити та підключитись до бази даних
conn = sqlite3.connect("titanic.db")
# SQLite тепер керує вашими даними!
```

## З CSV до бази даних

Пам'ятаєте урок 02? Ви робили це:

```python
import pandas as pd

titanic = pd.read_csv("Titanic Dataset.csv")
titanic.head()
```

Тепер давайте зберегти ці ж дані в базу даних:

```python
import pandas as pd
import sqlite3

# Завантажити CSV (ви вже це знаєте)
titanic = pd.read_csv("Titanic Dataset.csv")

# Зберегти в базу даних (нове!)
conn = sqlite3.connect("titanic.db")
titanic.to_sql("passengers", conn, if_exists="replace", index=False)
```

От і все. Тепер у вас є файл `titanic.db` з таблицею `passengers`.

## Знайомство з SQL

**SQL (Structured Query Language)** — це як ви розмовляєте з базою даних.

Це мова, як Python. Але замість того, щоб щось робити, SQL **ставить питання** про дані.

На уроці 02 ви ставили pandas питання:
```python
titanic.head()
titanic[["name", "age"]]
```

На уроці 03 ви ставите базі даних ті ж питання мовою SQL:
```sql
SELECT * FROM passengers LIMIT 5
SELECT name, age FROM passengers
```

---

## Ваш перший SQL запит

Давайте запустимо SQL запит у Python:

```python
import pandas as pd
import sqlite3

# Підключитись до бази даних
conn = sqlite3.connect("titanic.db")

# Запитати базу даних за допомогою SQL
result = pd.read_sql("SELECT * FROM passengers LIMIT 5", conn)
print(result)
```

`pd.read_sql()` запускає SQL запит і повертає результати як pandas DataFrame. Як раніше — але тепер дані походять з бази даних!

---

## SQL: SELECT та FROM

Кожен SQL запит має цю базову структуру:

```sql
SELECT що FROM звідки
```

**SELECT** = які колонки ви хочете
**FROM** = в якій таблиці вони знаходяться

Простий приклад:
```sql
SELECT name, age FROM passengers
```

Переклад: "Дай мені колонки `name` і `age` з таблиці `passengers`."

Щоб отримати всі колонки одразу, використовуйте `*` (символ підстановки):
```sql
SELECT * FROM passengers
```

## Дослідження структури вашої бази даних

Перш ніж писати запити, вам потрібно знати:
- Які таблиці існують?
- Які колонки в кожній таблиці?
- Який тип даних в кожній колонці?

SQLite має вбудований спосіб перевірки:

```python
# Див. все таблиці в базі даних
pd.read_sql("SELECT name FROM sqlite_master WHERE type='table'", conn)

# Див. назви колонок і типи для таблиці passengers
pd.read_sql("PRAGMA table_info(passengers)", conn)
```

Це як `.info()` і `.columns` у pandas — але для самої бази даних.

## Ключові концепції

| Концепція | Що це означає |
|---------|---|
| **Дані** | Сирі факти без контексту |
| **Інформація** | Дані організовані та задані значенням |
| **База даних** | Організована колекція пов'язаних даних |
| **DBMS** | Програмне забезпечення, яке керує базою даних |
| **SQLite** | Проста, файлова база даних |
| **SQL** | Мова для ставлення питань про дані |
| **`.to_sql()`** | Метод pandas для збереження DataFrame в базу даних |
| **`.read_sql()`** | Метод pandas для запуску SQL і отримання DataFrame |

---

## Ключові висновки

1. **Бази даних організовують дані краще, ніж CSV файли** — перевірка, без дублювання, безпечний доступ багатьох користувачів

2. **SQLite це реальна база даних** — можна створити за одну Python команду

3. **SQL це мова запитів** — як pandas, але для баз даних; ви ставите питання за допомогою SELECT та FROM

4. **Pandas і SQL працюють разом** — напишіть SQL в `.read_sql()`, отримайте результати як DataFrame
