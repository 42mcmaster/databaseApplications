## Ключова термінологія

| Термін | Визначення |

| **Дані (Data)** | Сирі факти без контексту (числа, текст, значення) |
| **Інформація (Information)** | Дані, які організовані та задані значенням |
| **База даних (Database)** | Організована колекція пов'язаних даних, керована програмним забезпеченням |

| **Рядок / Запис (Row / Record)** | Одиничний запис в таблиці (один пасажир, один студент, один товар) |

| **SQLite** | Легка, файлова реляційна база даних — сервер не потрібен |
| **Запит (Query)** | Запит для отримання даних з бази даних, написаний мовою SQL |
| **SELECT** | SQL ключове слово, яке специфікує, які колонки отримати |
| **FROM** | SQL ключове слово, яке специфікує, з якої таблиці запитувати |
| **PRAGMA** | SQLite команди для дослідження структури бази даних (таблиці, колонки, типи) |
| **Підключення (conn)** | Python об'єкт, який представляє відкритий зв'язок з файлом бази даних |
| **Цілісність даних (Data Integrity)** | Забезпечення того, що дані точні, послідовні та відповідають правилам |

## Огляд концепцій

### Що таке база даних?

База даних — це організована колекція пов'язаних даних. Замість того, щоб зберігати дані в окремих CSV файлах, бази даних зберігають дані в структурованих таблицях з правилами про те, які дані можна зберігати. Це дозволяє:
- **Запобігти поганим даним** — база даних перевіряє дані перед збереженням
- **Запобігти дублюванню** — зберігайте дані один раз, використовуйте скрізь
- **Безпечний спільний доступ** — багато людей можуть використовувати базу даних одночасно
- **Швидкі питання** — SQL робить пошук даних швидким

### Чому бази даних замість CSV?

CSV файли добре працюють для невеликих, простих наборів даних. Але в міру зростання даних CSV мають серйозні проблеми: немає правил для запобігання поганим даним, дані дублюються, а тільки одна людина може редагувати одночасно. Бази даних вирішують усі ці проблеми.

### DBMS (система управління базою даних)

DBMS — це програмний шар, який керує базою даних. Він зберігає дані на диску, забезпечує правила, керує безпекою та трансформує ваші SQL запити в операції. Ми використовуємо SQLite, тому що це просто (один файл, без сервера), безплатно і вбудовано в Python.

### Типи баз даних

Реляційні бази даних (які ми використовуємо) зберігають дані в таблицях, пов'язаних стосунками. NoSQL бази даних зберігають дані по-іншому і використовуються для соціальних мереж. Хмарні бази даних розташовані в Інтернеті та керуються компаніями на кшталт Amazon і Google.

### CSV до SQLite

Mosток від pandas до баз даних використовує два ключові методи: `to_sql()` зберігає DataFrame в таблицю бази даних, а `read_sql()` запускає SQL запит і повертає результати як DataFrame.

### Основи SQL

SQL — це як ви ставите питання базі даних. Найбільш базовий запит — це `SELECT колонки FROM таблиця`. Використовуйте `*` для отримання всіх колонок. На цьому курсі ми завжди запускаємо SQL через pandas за допомогою `pd.read_sql("запит", conn)`, щоб результати повертались як DataFrame.

### PRAGMA команди

PRAGMA специфічна для SQLite та дозволяє дослідити структуру вашої бази даних. `SELECT name FROM sqlite_master WHERE type='table'` показує всі таблиці. `PRAGMA table_info(tableName)` показує назви колонок і типи даних.

## Швидка карточка довідника

### Перетворення CSV на SQLite
```python
import pandas as pd
import sqlite3

titanic = pd.read_csv("Titanic Dataset.csv")
conn = sqlite3.connect("titanic.db")
titanic.to_sql("passengers", conn, if_exists="replace", index=False)
```

### Запуск SQL запитів
```python
pd.read_sql("SELECT * FROM passengers", conn)
pd.read_sql("SELECT name, age FROM passengers LIMIT 10", conn)
```

### Дослідження бази даних
```python
pd.read_sql("SELECT name FROM sqlite_master WHERE type='table'", conn)
pd.read_sql("PRAGMA table_info(passengers)", conn)
```

### Закриття підключення
```python
conn.close()
```
