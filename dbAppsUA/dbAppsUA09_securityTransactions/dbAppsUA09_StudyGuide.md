# Поясна записка: Безпека та Транзакції

## Основні терміни безпеки

1. **SQL-ін'єкція** (SQL injection) — атака, коли зловмисник вводить SQL-код через форму, щоб змінити запит
2. **Параметризований запит** (parameterized query) — запит з заповнювачами (?), де дані передаються окремо від коду
3. **Вразливість** (vulnerability) — слабкість у коді, яку можна експлуатувати
4. **Хешування** (hashing) — одностороння функція для безпечного збереження паролів
5. **Шифрування** (encryption) — перетворення даних у форму, яку не можна прочитати без ключа
6. **Контроль доступу** (access control) — обмеження того, що користувачі можуть робити
7. **Аудит** (audit log) — запис того, хто, коли й що змінив

## Терміни транзакцій

1. **Транзакція** (transaction) — група операцій, яка виконується як одне ціле
2. **COMMIT** — команда для збереження всіх змін транзакції
3. **ROLLBACK** — команда для скасування всіх змін транзакції
4. **BEGIN** — команда для запуску транзакції

## ACID властивості

1. **Atomicity (Атомарність)** — операція виконується повністю або зовсім не виконується; немає половинчастих результатів
2. **Consistency (Узгодженість)** — база даних переходить з одного коректного стану в інший; дані залишаються чесними
3. **Isolation (Ізоляція)** — одна транзакція не мешає іншій; зміни видимі лише після COMMIT
4. **Durability (Надійність)** — один раз збережено (COMMIT) — дані безпечні назавжди, навіть при збоях системи

## SQL-ін'єкція: Швидка довідка

### Небезпечний код:
```python
username = input("Username: ")
password = input("Password: ")
query = "SELECT * FROM users WHERE username = '" + username + "' AND password = '" + password + "'"
cursor.execute(query)
```

### Безпечний код:
```python
username = input("Username: ")
password = input("Password: ")
query = "SELECT * FROM users WHERE username = ? AND password = ?"
cursor.execute(query, (username, password))
```

**Ключова різниця:** параметризовані запити передають дані окремо від структури SQL.

## Транзакції: Синтаксис та приклад

### Синтаксис:
```sql
BEGIN;
-- операції SQL
COMMIT;
-- або
ROLLBACK;
```

### Приклад банку:
```sql
BEGIN;
UPDATE accounts SET balance = balance - 100 WHERE user_id = 1;
UPDATE accounts SET balance = balance + 100 WHERE user_id = 2;
COMMIT;
```

Якщо система впаде між операціями, обидві скасовуються автоматично.

## Визначення ACID

| Властивість | Визначення | Приклад |
|------------|-----------|--------|
| **Atomicity** | Операція — все або нічого | При збої транзакція скасовується повністю |
| **Consistency** | Дані залишаються коректними | Сума грошей в банку не змінюється при переводі |
| **Isolation** | Транзакції не мешають одна одній | Две транзакції можуть виконуватись паралельно |
| **Durability** | Збережені дані залишаються безпечні | Резервні копії захищають від апаратних збоїв |

## Поширені помилки

1. ❌ Конкатенація введення користувача в SQL-запити

4. ❌ Ігнорування прав доступу користувачів
5. ❌ Відсутність резервних копій

## Питання для повторення

1. **Що таке SQL-ін'єкція і як вона працює?**
   Це атака, коли зловмисник вводить SQL-код через форму. Система виконує код замість того, щоб трактувати введення як дані.

2. **Як запобігти SQL-ін'єкції?**
   Використовуйте параметризовані запити з заповнювачами (?).

3. **Коли потрібні транзакції?**
   Коли операції залежать одна від одної і повинні виконуватись разом (наприклад, переведення грошей).

4. **Що таке ACID?**
   Чотири властивості надійних систем баз даних: Atomicity, Consistency, Isolation, Durability.

5. **Яка відповідальність розробника щодо захисту даних?**
   Дотримуватись законів про захист персональних даних, використовувати параметризовані запити, хешувати паролі, логувати дії, робити резервні копії.

6. **Чим різняться COMMIT і ROLLBACK?**
   COMMIT зберігає всі зміни транзакції; ROLLBACK скасовує їх.
