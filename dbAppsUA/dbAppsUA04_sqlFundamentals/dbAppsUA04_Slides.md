## Як працює SQL

Уявіть базу даних як величезну бібліотеку. SQL — це як запитувати бібліотекаря:

- "Покажи мені всі книжки про детективів" → `SELECT`
- "Шукай тільки у розділі художної літератури" → `FROM`
- "Пропусти видані до 2020 року" → `WHERE`

Кожний SQL запит має просту структуру:

```sql
SELECT стовпці_які_ви_хочете
FROM назва_таблиці
WHERE умова
```

---

## Ваш перший запит: структура

```sql
SELECT name, age
FROM passengers
WHERE age > 30
```

**Що робить цей запит:**
- Вибирає стовпці `name` та `age`
- З таблиці `passengers`
- Тільки для людей старших 30 років

Результат: менша таблиця тільки з цими рядками.

---

## Запуск SQL у Python

```python
import pandas as pd
import sqlite3

conn = sqlite3.connect("titanic.db")
result = pd.read_sql(
    "SELECT name, age FROM passengers WHERE age > 30",
    conn
)
print(result)
```

> **Ключева ідея:** SQL запити завжди повертають результат, з яким ви можете працювати в pandas.

## Фільтрування з операторами порівняння (Comparison Operators)

`WHERE` як охоронець — воно зберігає тільки рядки, де умова дорівнює True.

| Оператор | Значення | Приклад |
|----------|----------|---------|
| `=` | Дорівнює | `WHERE pclass = 1` |
| `!=` | Не дорівнює | `WHERE sex != 'male'` |
| `>` | Більше ніж | `WHERE age > 30` |
| `<` | Менше ніж | `WHERE fare < 10` |
| `>=` | Більше або дорівнює | `WHERE age >= 18` |
| `<=` | Менше або дорівнює | `WHERE pclass <= 2` |

**Порада:** текстові значення беруть у одинарні лапки: `'female'`, не `"female"`

---

## Об'єднання умов: AND, OR, NOT

**AND** — обидві умови повинні бути правильні:
```sql
SELECT name, sex, age FROM passengers
WHERE sex = 'female' AND age < 18
```

**OR** — хоча б одна умова може бути правильна:
```sql
SELECT name, pclass FROM passengers
WHERE pclass = 1 OR pclass = 2
```

**NOT** — змінює умову:
```sql
SELECT name, pclass FROM passengers
WHERE NOT pclass = 3
```

---

## Таблиця логічних операторів (Boolean Logic)

Ці оператори працюють так само, як `&` та `|`, які ви використовували в pandas:

| Оператор | Значення | Приклад |
|----------|----------|---------|
| `AND` | Обидві мають бути правильні | `age > 18 AND sex = 'female'` |
| `OR` | Хоча б одна може бути правильна | `pclass = 1 OR pclass = 2` |
| `NOT` | Змінює умову | `NOT survived = 1` |

> **Пам'ятайте:** ви можете об'єднувати кілька умов. Комп'ютер оцінює їх зліва направо.

## Пошук за шаблоном: LIKE

Використовуйте **LIKE** для пошуку текстових шаблонів. Символ `%` означає "будь-які символи":

```sql
-- Знайти всі імена, що містять "Mrs"
SELECT name FROM passengers WHERE name LIKE '%Mrs%'

-- Знайти імена, що починаються з "Mr"
SELECT name FROM passengers WHERE name LIKE 'Mr%'
```

Це дуже потужно для реальних даних!

---

## Фільтрування діапазонів: BETWEEN

Використовуйте **BETWEEN** для пошуку значень у діапазоні (включаючи межі):

```sql
SELECT name, age
FROM passengers
WHERE age BETWEEN 18 AND 30
```

Це чистіше, ніж писати: `WHERE age >= 18 AND age <= 30`

---

## Пошук у списку: IN

Використовуйте **IN**, коли ви хочете знайти будь-яке значення зі списку:

```sql
SELECT name, embarked
FROM passengers
WHERE embarked IN ('C', 'Q')
```

Набагато легше, ніж: `WHERE embarked = 'C' OR embarked = 'Q' OR embarked = 'S'`

---

## Три скорочені оператори

| Оператор | Мета | Приклад |
|----------|------|---------|
| `LIKE` | Пошук за шаблоном | `name LIKE '%Smith%'` |
| `BETWEEN` | Діапазон значень | `age BETWEEN 20 AND 40` |
| `IN` | Пошук у списку | `port IN ('C', 'Q')` |

> **Порада:** ці оператори роблять запити легшими для читання та швидшими для написання.

---

## Сортування результатів: ORDER BY

Використовуйте **ORDER BY** для впорядкування ваших результатів:

```sql
-- Сортування за віком, наймолодший першим (ASC = за зростанням)
SELECT name, age FROM passengers ORDER BY age ASC

-- Сортування за віком, найстаріший першим (DESC = за спаданням)
SELECT name, age FROM passengers ORDER BY age DESC
```

---

## Обмеження результатів: LIMIT

Використовуйте **LIMIT** для отримання тільки перших N рядків:

```sql
-- 5 найдорожчих квитків
SELECT name, fare
FROM passengers
ORDER BY fare DESC
LIMIT 5
```

Це дуже корисно, коли ви працюєте з величезними наборами даних!

## Повний запит: всі частини разом

```sql
SELECT name, age, fare, survived
```

**Читайте це так:**

3. "Тільки рядки де pclass = 1 AND age >= 18"
4. "Сортуй за fare (найбільше першим)"
5. "Покажи мені тільки перші 10"

## Правило порядку клаузул (Clauses)

**Цей порядок ЗАВЖДИ важливий у SQL:**

```
SELECT → FROM → WHERE → ORDER BY → LIMIT
```

**Що → Звідки → Фільтр → Сортування → Скільки**

Якщо ви помістите їх у неправильному порядку, ваш запит не працюватиме!

## Практика: усе разом

Знайдіть 5 найдорожчих квитків, куплених жіночими пасажирами старші 25 років:

```sql
SELECT name, age, fare
FROM passengers
WHERE sex = 'female' AND age > 25
ORDER BY fare DESC
LIMIT 5
```

**Розберіть це:**
- Що: `name, age, fare`
- Звідки: `passengers`
- Фільтр: жіночі та старші 25
- Сортування: найдорожчі першим
- Обмеження: 5 рядків

---

## Ключові висновки

1. **Структура SQL завжди одна й та сама:** SELECT → FROM → WHERE → ORDER BY → LIMIT

2. **Оператори порівняння** (`=`, `>`, `<`, тощо) фільтрують за умовами

3. **Логічні оператори** (AND, OR, NOT) дозволяють вам об'єднувати кілька умов

4. **Скорочені оператори** (LIKE, BETWEEN, IN) роблять фільтрування легшим

5. **ORDER BY та LIMIT** дозволяють вам сортувати та контролювати кількість повернених рядків
