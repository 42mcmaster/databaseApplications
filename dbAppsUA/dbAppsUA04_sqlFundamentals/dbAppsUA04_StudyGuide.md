| Термін | Визначення |

## Огляд концепцій

### SELECT та FROM

Кожний SQL запит починається з SELECT (які стовпці) та FROM (яка таблиця). Використовуйте `*` для всіх стовпців або список конкретних назв стовпців, розділених комами. Назви стовпців повинні точно відповідати таблиці. SQL не чутлива до регістру для ключових слів (SELECT = select), але назви стовпців повинні точно співпадати.

### WHERE та оператори порівняння

WHERE фільтрує рядки. У результати включаються тільки рядки, де умова дорівнює True. Шість операторів порівняння: `=` (дорівнює), `!=` або `<>` (не дорівнює), `>` (більше), `<` (менше), `>=` (більше або дорівнює), `<=` (менше або дорівнює). Текстові значення повинні бути взяті в одинарні лапки: `WHERE sex = 'female'`.

### Логічні оператори

Логічні оператори дозволяють об'єднувати умови: AND (обидві повинні бути правильні), OR (хоча б одна повинна бути правильна), NOT (змінює результат). Це ті ж концепції, які ви використовували в pandas з `&`, `|` та `~`.

### LIKE, BETWEEN, IN

LIKE здійснює пошук за шаблонами з `%` як підстановочний символ. `WHERE name LIKE '%Mrs.%'` знаходить будь-яке ім'я, що містить "Mrs.". BETWEEN фільтрує діапазон (включаючи межі): `WHERE age BETWEEN 18 AND 30` включає обидва 18 та 30. IN пошукує в списку: `WHERE embarked IN ('C', 'Q')` знаходить C або Q.

### ORDER BY та LIMIT

ORDER BY сортує результати. ASC (за зростанням) — за замовчуванням. DESC (за спаданням) потрібно вказати. LIMIT обмежує кількість повернених рядків. Вони завжди приходять після WHERE у запиті.

### Порядок клаузул

SQL вимагає клаузули в певному порядку: `SELECT → FROM → WHERE → ORDER BY → LIMIT`. Вам не потрібно використовувати кожну клаузулу, але ті, які ви використовуєте, повинні бути в цьому порядку.

## SQL Швидкий довідник

### Базові запити
```sql
SELECT * FROM passengers                         -- Всі стовпці, всі рядки
SELECT name, age FROM passengers                  -- Конкретні стовпці
SELECT name, age FROM passengers LIMIT 10         -- Перші 10 рядків
```

### Фільтрування за допомогою WHERE
```sql
SELECT * FROM passengers WHERE pclass = 1         -- Тільки перший клас
SELECT * FROM passengers WHERE age > 30           -- Старші 30 років
SELECT * FROM passengers WHERE sex = 'female'     -- Тільки жіночі
SELECT * FROM passengers WHERE fare >= 50         -- Вартість мінімум 50
```

### Об'єднання умов
```sql
-- AND: обидві повинні бути правильні
SELECT * FROM passengers WHERE sex = 'female' AND survived = 1

-- OR: хоча б одна може бути правильна
SELECT * FROM passengers WHERE pclass = 1 OR pclass = 2

-- NOT: змінює умову
SELECT * FROM passengers WHERE NOT pclass = 3
```

### Пошук за шаблонами та діапазони
```sql
-- LIKE: пошук за шаблонами
SELECT * FROM passengers WHERE name LIKE '%Dr.%'
SELECT * FROM passengers WHERE name LIKE 'A%'      -- Імена, що починаються з A

-- BETWEEN: діапазон значень (включно)
SELECT * FROM passengers WHERE age BETWEEN 20 AND 40

-- IN: пошук у списку
SELECT * FROM passengers WHERE embarked IN ('C', 'Q')
SELECT * FROM passengers WHERE pclass IN (1, 2)
```

### Сортування та обмеження
```sql
-- Сортування за зростанням (за замовчуванням)
SELECT name, age FROM passengers ORDER BY age

-- Сортування за спаданням
SELECT name, fare FROM passengers ORDER BY fare DESC

-- Об'єднайте все разом
SELECT name, age, fare
FROM passengers
WHERE pclass = 1 AND survived = 1
ORDER BY fare DESC
LIMIT 10
```

## Довідник логічних операторів (Boolean Logic)

### Оператори порівняння
| Оператор | Значення | Приклад |
|----------|----------|---------|
| `=` | Дорівнює | `age = 30` |
| `!=` або `<>` | Не дорівнює | `sex != 'male'` |
| `>` | Більше ніж | `age > 30` |
| `<` | Менше ніж | `fare < 10` |
| `>=` | Більше або дорівнює | `age >= 18` |
| `<=` | Менше або дорівнює | `pclass <= 2` |

### Логічні оператори
| Оператор | Значення | Приклад |
|----------|----------|---------|
| `AND` | Обидві умови правильні | `age > 18 AND sex = 'female'` |
| `OR` | Хоча б одна умова правильна | `pclass = 1 OR pclass = 2` |
| `NOT` | Змінює умову | `NOT survived = 1` |
