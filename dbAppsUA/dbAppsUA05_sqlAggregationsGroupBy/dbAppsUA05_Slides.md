## Що таке агрегуючі функції?

**Агрегуюча функція** (aggregate function) бере багато рядків і обчислює **одне єдине значення**.

Подумайте про це як про калькулятор:
- **COUNT(*)** → Скільки рядків?
- **SUM()** → Додай все разом
- **AVG()** → Яке середнє значення?
- **MAX()** → Який найбільший?
- **MIN()** → Який найменший?

---

## Поширені агрегуючі функції

```sql
SELECT COUNT(*) FROM team_game_stats
  -- Результат: 10842 (всього ігор зіграно)

SELECT AVG(pts) FROM player_season_stats
  -- Результат: 8.2 (середні очки за гравця за сезон)

SELECT SUM(pts) FROM team_game_stats
  -- Результат: 8934201 (всього очків у всіх іграх)

SELECT MAX(pts) FROM team_game_stats
  -- Результат: 156 (найвищий рахунок гри)
```

---

## Округлення результатів

Реальні дані грубі. Функція `ROUND()` їх очищує:

```sql
SELECT ROUND(AVG(pts), 1) FROM player_season_stats
  -- Результат: 8.2 (округлено до 1 знака після коми)

SELECT ROUND(AVG(pts), 0) FROM player_season_stats
  -- Результат: 8 (округлено до найближчого цілого)
```

> **Резюме:** Агрегуючі функції складають багато рядків в один розрахунок. Використовуйте їх, щоб відповісти на питання типу "Скільки?", "Яка сума?" або "Яке середнє?"

## Вступ до GROUP BY

Що якщо вам не потрібна **одна сума** — вам потрібні суми **для кожної команди** або **за сезон**?

Саме для цього існує **GROUP BY**. Він розділяє дані на групи і обчислює агрегат **для кожної групи**.

**Без GROUP BY:**
```sql
SELECT COUNT(*) FROM team_game_stats
  -- Результат: 10842 (одне велике число)
```

**З GROUP BY:**
```sql
SELECT team_id, COUNT(*) FROM team_game_stats
GROUP BY team_id
  -- Результат: 30 рядків (один лічильник для кожної команди)
```

---

## Приклад GROUP BY: Ігри на команду

```sql
SELECT team_id, COUNT(*) AS games
FROM team_game_stats
GROUP BY team_id
```

| team_id | games |
|---------|-------|
| BOS     | 362   |
| LAL     | 361   |
| MIA     | 358   |
| ...     | ...   |

Результат містить **один рядок на команду** з лічильником для кожної.

---

## GROUP BY з кількома колонками

Можете групувати більш ніж за однією колонкою:

```sql
SELECT team_id, season, COUNT(*) AS games
FROM team_game_stats
GROUP BY team_id, season
```

Тепер ви отримуєте **один рядок на кожну комбінацію команда-сезон** — розбиття за командою І роком.

---

## Зроблення результатів читабельними з Aliases (AS)

Коли використовуєте агрегуючі функції, імена колонок стають потворними:

```sql
SELECT team_id, COUNT(*), ROUND(AVG(pts), 1)
FROM team_game_stats
GROUP BY team_id
```

Колонки результату названі: `team_id`, `COUNT(*)`, `ROUND(AVG(pts), 1)` — не корисно.

Використовуйте **AS** для їх перейменування:

```sql
SELECT team_id,
       COUNT(*) AS totalGames,
       ROUND(AVG(pts), 1) AS avgPointsPerGame
FROM team_game_stats
GROUP BY team_id
```

Тепер колонки ясні та читабельні.

## WHERE vs HAVING: Знайте різницю

Обидва фільтрують дані, але у різний час:

| | WHERE | HAVING |
|--|-------|--------|
| **Фільтрує** | Окремі рядки | Групи (після GROUP BY) |
| **Час** | Перед групуванням | Після групування |
| **Можна агрегати?** | Ні | Так |

**WHERE** звужує сирові дані **спочатку**, потім GROUP BY узагальнює менший датасет. **HAVING** працює **після** групування — фільтрує рядки резюме, не сирові дані.

```sql
-- Приклад WHERE: фільтрувати перед групуванням
SELECT team_id, AVG(pts) AS avgPts
FROM team_game_stats
WHERE season = '2023-24'
GROUP BY team_id

-- Приклад HAVING: фільтрувати групи після групування
SELECT team_id, COUNT(*) AS games, AVG(pts) AS avgPts
FROM team_game_stats
GROUP BY team_id
HAVING AVG(pts) > 110
```

> **Резюме:** Використовуйте WHERE для сирих даних. Використовуйте HAVING для результатів резюме.

## Повний порядок запиту

Коли пишете запит, речення повинні з'являтися в точному порядку:

```sql
SELECT column, AGG(column) AS alias
FROM table
WHERE condition              -- фільтрувати рядки спочатку
GROUP BY column              -- розділити на групи
HAVING AGG(column) > value   -- фільтрувати групи
ORDER BY alias DESC          -- сортувати результати
LIMIT 10                     -- обмежити вихід
```

**Запам'ятайте:** SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT

## Експорт результатів запиту в Excel

Запуск запитів це чудово, але іноді потрібно **поділитися результатами в Excel**.

```python
import pandas as pd, sqlite3

# Крок 1: Підключіться та запустіть запит
conn = sqlite3.connect("nba_5seasons.db")
topScorers = pd.read_sql("""
    SELECT full_name, pts, season
    FROM player_season_stats
    ORDER BY pts DESC
    LIMIT 20
""", conn)

# Крок 2: Збережіть в Excel
topScorers.to_excel("top_scorers.xlsx", index=False)
```

Це все — результати SQL тепер в Excel.

## Ключові висновки

1. **Агрегуючі функції** (COUNT, AVG, SUM, MAX, MIN) обчислюють резюме з багатьох рядків
2. **GROUP BY** розділяє дані на групи і обчислює одне резюме для кожної групи
3. **Aliases (AS)** роблять імена колонок результату ясними та читабельними
4. **WHERE** фільтрує сирі рядки; **HAVING** фільтрує узагальнені групи
5. **Порядок речень важливий:** SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT
6. **Експортуйте в Excel** з pandas для професійних звітів
