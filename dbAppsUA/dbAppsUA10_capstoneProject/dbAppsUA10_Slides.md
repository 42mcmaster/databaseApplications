# Фаза 1: Дизайн

## Визначте сутності та атрибути

**Сутність** — об'єкт (напр., Книга, Автор)
**Атрибут** — властивість (напр., название, рік видання)

**Приклад: Бібліотека**

| Сутність | Атрибути |
|---------|----------|
| Книга | ID, назва, автор_ID, рік видання, жанр |
| Автор | ID, ім'я, країна, дата народження |

---

## Визначте зв'язки

**Зв'язок** (relationship) — як таблиці з'єднуються

| Тип | Приклад |
|-----|---------|
| 1 до N | Один автор → Багато книг |
| N до N | Багато студентів ↔ Багато курсів |

---

## Приклад: Притулок для тварин

```
Притулок
├── Тварина (ID, ім'я, вид, дата народження)
│   └── притулок_ID → Притулок
└── Опікун (ID, ім'я, телефон)
    └── тварина_ID → Тварина
```

---

## Створіть ER-діаграму та CREATE TABLE

**ER-діаграма:**
```
┌─────────┐        ┌────────────┐
│ Тварина │────1:N │  Опікун    │
│ (ID, ім'я)      │ (ID, ім'я) │
└─────────┘        └────────────┘
```

**CREATE TABLE:**
```sql
CREATE TABLE animals (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  species TEXT,
  birth_date DATE
);

CREATE TABLE guardians (
  id INTEGER PRIMARY KEY,
  name TEXT NOT NULL,
  phone TEXT,
  animal_id INTEGER,
  FOREIGN KEY (animal_id) REFERENCES animals(id)
);
```

# Фаза 3: Запити

## Напишіть 8+ запитів різного типу

Приклади:

1. **Простий SELECT**
   ```sql
   SELECT name FROM animals WHERE species = 'собака';
   ```

2. **Агреговані (COUNT, SUM, AVG)**
   ```sql
   SELECT COUNT(*) FROM animals;
   SELECT species, COUNT(*) FROM animals GROUP BY species;
   ```

3. **JOIN (з'єднання таблиць)**
   ```sql
   SELECT animals.name, guardians.name
   FROM animals
   JOIN guardians ON animals.id = guardians.animal_id;
   ```

---

## Можливі запити для вашого проекту

- Скільки записів в кожній таблиці?
- Назвіть все, що відповідає критерію (WHERE)

- Сортуйте за значенням (ORDER BY)
- Знайдіть максимум/мінімум (MAX, MIN)
- Обчисліть середнє (AVG)

| guardians | animal_id | INTEGER | зовнішній ключ до animals(id) |
