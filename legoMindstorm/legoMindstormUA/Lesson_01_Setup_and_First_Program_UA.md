# Урок 01 – Налаштування та перша програма
### EV3 MicroPython Робототехніка

---

## Що ви навчитесь
До кінця цього уроку ви:
- Встановите розширення EV3 MicroPython у VS Code
- Зрозумієте, як ваш комп'ютер і EV3 brick працюють разом
- Створите та запустите свою першу програму на MicroPython для EV3

**Час:** ~45 хвилин
**Що вам потрібно:** EV3 Brick (з уже вставленою SD-карткою), кабель Mini-USB, комп'ютер із VS Code

---

## Загальна картина – Як це працює?

Перш ніж щось торкати, ознайомтеся із загальною картиною:

```
Ваш комп'ютер                          EV3 Brick
--------------                         ----------
Ви пишете код  →  USB кабель  →  Код виконується тут
у VS Code
```

У вашому EV3 brick збоку вставлена **microSD-картка**. Ця картка замінює стандартне програмне забезпечення LEGO на **MicroPython** — версію Python, розроблену для запуску на маленьких комп'ютерах і роботах. Ви пишете код на комп'ютері у VS Code, надсилаєте його на brick через USB, і він виконується.

> **MicroSD-картка повинна завжди залишатися в EV3.** Якщо її вийняти, brick повернеться до оригінальної прошивки LEGO і MicroPython не працюватиме.

---

## Частина 1 – Встановлення розширення EV3 MicroPython

VS Code вже встановлений на вашому комп'ютері, але нам потрібно додати одне розширення, щоб він зміг спілкуватися з EV3 brick.

1. Відкрийте **Visual Studio Code**
2. Натисніть на іконку **Extensions** у лівій бічній панелі (схожа на 4 квадрати)
3. У рядку пошуку введіть: `EV3 MicroPython`
4. Знайдіть **"LEGO® MINDSTORMS® EV3 MicroPython"** від LEGO Education
5. Натисніть **Install**

Коли встановлення завершиться, у лівій бічній панелі з'явиться маленька іконка brick. Це панель EV3 — ви будете використовувати її щоразу, коли починаєте новий проєкт.

---

## Частина 2 – Увімкнення та підключення EV3 Brick

1. Натисніть **темно-сіру центральну кнопку**, щоб увімкнути EV3
2. Зачекайте, поки він повністю завантажиться — це займає **1–2 хвилини**
3. Ви дізнаєтеся, що він готовий, коли **індикатор стане зеленим**
4. Підключіть **кабель mini-USB** до верхньої частини EV3, а потім до комп'ютера

> **Повільне завантаження — це нормально.** Під час запуску на екрані EV3 буде прокручуватися багато тексту. Просто зачекайте зеленого світла, перш ніж щось робити.

---

## Частина 3 – Створення першого проєкту

У MicroPython ваш код зберігається у **папці проєкту**. У кожному проєкті є файл `main.py` — саме цей файл EV3 запускає, коли ви стартуєте програму.

```
getting_started/     ← папка вашого проєкту
  └── main.py        ← сюди пишіть код
```

### Кроки:

1. У VS Code натисніть на іконку **EV3 MicroPython** (іконка brick у лівій бічній панелі)
2. Натисніть **"Create a new project"**
3. Введіть `getting_started` як назву проєкту і натисніть **Enter**
4. Виберіть місце для збереження (Робочий стіл або Документи підійдуть) і натисніть **Choose Folder**

VS Code створить проєкт і автоматично відкриє `main.py`.

---

## Частина 4 – Читання стартового коду

Перш ніж щось писати, подивимося, що вже є у `main.py`. Ви вже бачили Python раніше, але деякі речі тут будуть новими — це нормально. Ми розберемо кожен рядок.

```python
#!/usr/bin/env pybricks-micropython
# Цей перший рядок ОБОВ'ЯЗКОВО повинен бути на початку кожної програми.
# Він повідомляє EV3 brick, що цей файл є програмою MicroPython.

from pybricks.hubs import EV3Brick
# Завантажує клас EV3Brick — дає нам контроль над екраном,
# динаміком, кнопками та індикатором brick.

from pybricks.ev3devices import (Motor, TouchSensor, ColorSensor,
                                  InfraredSensor, UltrasonicSensor, GyroSensor)
# Завантажує всі мотори та датчики, які ми можемо використовувати.
# Ми не використаємо все це сьогодні — вони просто готові на майбутнє.

from pybricks.parameters import Port, Stop, Direction, Button, Color
# Завантажує константи — наприклад, Port.B (до якого порту підключений мотор)
# або Color.RED. Думайте про це як про спільний словник для робота.

from pybricks.tools import wait, StopWatch, DataLog
# wait()     → зупиняє програму на певну кількість мілісекунд
# StopWatch  → таймер
# DataLog    → зберігає дані у файл (корисно для експериментів пізніше)

from pybricks.robotics import DriveBase
# DriveBase спрощує керування двоколісним роботом як єдиним цілим.
# Ми використаємо це в наступних уроках.

# -------------------------------------------------------
# Створюйте свої об'єкти тут
# -------------------------------------------------------
ev3 = EV3Brick()
# Це створює об'єкт EV3Brick і зберігає його у змінній ev3.
# Відтепер ev3.speaker, ev3.screen, ev3.light — все працює.

# -------------------------------------------------------
# Пишіть свою програму тут
# -------------------------------------------------------
ev3.speaker.beep()
# Змушує EV3 відтворити короткий звуковий сигнал. Ось і весь стартовий код!
```

Кілька речей, на які варто звернути увагу:
- Рядки, що починаються з `#`, — це **коментарі** — EV3 повністю їх ігнорує. Це нотатки для програміста.
- `from pybricks.X import Y` — це той самий синтаксис `import`, який ви вже бачили в Python — ми просто підключаємо інструменти, специфічні для EV3.
- `ev3 = EV3Brick()` створює **об'єкт**. Ви будете використовувати `ev3.щось` для керування майже всім на brick.

> **Вам не потрібно запам'ятовувати імпорти.** Кожен новий проєкт починається з них автоматично. Протягом наступних уроків ви дізнаєтеся, що кожен із них робить, коли будете його використовувати.

## Частина 5 – Підключення та запуск стартової програми

Давайте перевіримо, що все працює, перш ніж писати власний код.

### Підключення до brick:

1. У VS Code знайдіть внизу лівої бічної панелі розділ **"EV3DEV DEVICE BROWSER"**
2. Натисніть **"Click here to connect to a device"**
3. Ваш EV3 повинен з'явитися у списку — натисніть на нього, щоб підключитися
4. Коли підключення встановлено, поруч із назвою brick ви побачите зелену крапку

### Запуск:

Натисніть **F5** (або перейдіть до **Run → Start Debugging**).

VS Code скопіює ваші файли на EV3 і запустить програму. Ви повинні почути **короткий звуковий сигнал** з brick.

**Нічого не сталося?** Перевірте:
- USB підключений з обох кінців
- Індикатор EV3 зелений (не все ще завантажується)
- Ви натиснули на EV3 у браузері пристроїв і бачите зелену крапку

> **Щоб зупинити програму достроково:** Натисніть червоний квадрат **Stop** на панелі інструментів VS Code або натисніть **кнопку «назад»** на EV3 brick.

## Частина 6 – Ваша перша власна програма

Тепер давайте напишемо щось своє. Замініть **усе** у `main.py` кодом нижче. Читайте кожен коментар — вони пояснюють, що робить кожен рядок.

```python
#!/usr/bin/env pybricks-micropython

from pybricks.hubs import EV3Brick
from pybricks.parameters import Color
from pybricks.tools import wait

# Створюємо об'єкт EV3Brick. Він потрібен у кожній програмі.
ev3 = EV3Brick()

# -------------------------------------------------------
# ЕКРАН
# -------------------------------------------------------

# Очищаємо екран, щоб він був порожнім перед тим, як щось писати.
ev3.screen.clear()

# Друкуємо рядки тексту на дисплеї EV3.
ev3.screen.print("Hello, World!")
ev3.screen.print("EV3 is ready.")

# -------------------------------------------------------
# ІНДИКАТОР СТАНУ
# -------------------------------------------------------

# Змінюємо колір індикатора стану.
# Варіанти: Color.GREEN, Color.RED, Color.ORANGE
ev3.light.on(Color.ORANGE)

# -------------------------------------------------------
# ДИНАМІК
# -------------------------------------------------------

# beep(frequency, duration)
#   frequency → висота тону в Гц. Більше число = вищий тон.
#               440 Гц — музична нота A4 (стандартна настройна висота тону).
#   duration  → як довго лунає сигнал у мілісекундах.
#               500 мс = половина секунди.
ev3.speaker.beep(440, 500)

# wait() зупиняє програму на задану кількість мілісекунд.
# 1000 мс = 1 секунда.
wait(1000)

# Відтворюємо другий сигнал з вищим тоном.
ev3.speaker.beep(880, 500)   # 880 Гц — рівно на октаву вище, ніж 440 Гц

wait(1000)

# -------------------------------------------------------
# ЗАВЕРШЕННЯ
# -------------------------------------------------------

# Вмикаємо зелене світло, щоб сигналізувати про успішне завершення програми.
ev3.light.on(Color.GREEN)

ev3.screen.print("Done!")

# Чекаємо 2 секунди, щоб встигнути прочитати екран до виходу з програми.
wait(2000)
```

Збережіть файл (`Ctrl+S` у Windows / `Cmd+S` на Mac) і натисніть **F5**.

**Що ви повинні побачити і почути:**
- "Hello, World!" і "EV3 is ready." з'являються на екрані EV3
- Індикатор стану стає **помаранчевим**
- Лунають два сигнали — другий вищий
- Індикатор стану стає **зеленим**
- "Done!" з'являється на екрані

## Довідкова картка

| Код | Що робить |
|---|---|
| `#!/usr/bin/env pybricks-micropython` | Обов'язковий перший рядок — ідентифікує тип програми |
| `from pybricks.X import Y` | Завантажує інструменти з бібліотеки pybricks |
| `ev3 = EV3Brick()` | Створює об'єкт brick для керування ним |
| `ev3.screen.clear()` | Очищає дисплей EV3 |
| `ev3.screen.print("текст")` | Друкує рядок тексту на дисплеї |
| `ev3.light.on(Color.X)` | Змінює колір індикатора стану |
| `ev3.speaker.beep(freq, ms)` | Відтворює сигнал із заданою висотою тону та тривалістю |
| `wait(ms)` | Зупиняє програму (1000 мс = 1 секунда) |
| `# коментар` | Нотатка в коді — ігнорується EV3 |

## Словник

**MicroPython** – Версія Python, створена для запуску на маленьких пристроях, наприклад роботах.

**Project (Проєкт)** – Папка, що містить `main.py` та будь-які інші файли, потрібні вашій програмі.

**Object (Об'єкт)** – Змінна, яка представляє реальний фізичний предмет (наприклад, EV3 brick). Вона має власні вбудовані дії, що називаються **методами**.

**Method (Метод)** – Дія, що належить об'єкту. `ev3.speaker.beep()` викликає метод `beep` на `speaker`, який належить `ev3`.

**Milliseconds / мілісекунди (ms)** – Одиниця вимірювання часу в MicroPython. 1000 мс = 1 секунда.

## Усунення несправностей

| Проблема | Спробуйте це |
|---|---|
| EV3 не вмикається | Перевірте батареї |
| Індикатор стану помаранчевий і блимає | Ще завантажується — чекайте на зелений |
| VS Code не знаходить EV3 | Перевірте обидва кінці USB-кабелю; спробуйте інший USB-порт |
| Програма запущена, але немає звуку | Гучність EV3 може бути знижена — перевірте меню налаштувань brick |
| Червоне підкреслення в коді | Синтаксична помилка — шукайте друкарську помилку поблизу підкреслення |
| Програма аварійно завершується з помилкою | Прочитайте повідомлення на панелі **OUTPUT** внизу VS Code |
