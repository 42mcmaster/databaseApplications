## Довідка – Що таке Датчик Дотику?

Датчик дотику є найпростішим датчиком у наборі. На передній панелі є червона кнопка, і він повідомляє одне з двох: натискається кнопка прямо зараз чи ні?

```
Датчик Дотику
   ┌───────┐
   │   ●   │  ← червона кнопка
   └───────┘
   натиснуто   = True
   відпущено   = False
```

**Ключові факти:**
- Повідомляє **True** (натиснуто) або **False** (не натиснуто) — це все
- **Удар** означає натиснуто, а потім відпущено
- Підключається до будь-якого порту датчика (S1–S4)
- Ідеально підходить для запуску програм, зупинки програм або підрахунку подій

## Частина 1 – Налаштування та Перше Читання

Підключіть датчик дотику до **порту 1** на цеглині EV3. Встановіть його на своєму роботі в місці, куди ви легко можете дістатися — передній бампер є гарним місцем, якщо ви хочете, щоб робот виявляв зіткнення.

Створіть новий проект під назвою `touch_test`. Введіть цей код:

```python
#!/usr/bin/env pybricks-micropython

from pybricks.hubs import EV3Brick
from pybricks.ev3devices import TouchSensor
from pybricks.parameters import Port
from pybricks.tools import wait

ev3 = EV3Brick()

# Створіть об'єкт датчика дотику на порту 1.
bumper = TouchSensor(Port.S1)

ev3.speaker.beep()
ev3.screen.clear()

# Прочитайте датчик 20 разів, двічі на секунду.
for i in range(20):
    if bumper.pressed():
        ev3.screen.print("НАТИСНУТО")
    else:
        ev3.screen.print("відпущено")
    wait(500)
```

Запустіть його, а потім натискайте й відпускайте кнопку під час роботи. Дивіться, як змінюється екран.

**Нові команди:**

| Команда | Що вона робить |
|---|---|
| `TouchSensor(Port.S1)` | Створює об'єкт датчика дотику на порту 1 |
| `bumper.pressed()` | Повертає `True`, якщо натиснуто, `False`, якщо ні |

## Частина 2 – Реакція на Дотик

Тепер давайте зробимо так, щоб робот щось робив, коли натисна кнопка. Ось приклад, який чекає натиснення перед їздою — датчик дотику стає кнопкою "go".

```python
#!/usr/bin/env pybricks-micropython

from pybricks.hubs import EV3Brick
from pybricks.ev3devices import Motor, TouchSensor
from pybricks.parameters import Port
from pybricks.robotics import DriveBase
from pybricks.tools import wait

ev3 = EV3Brick()
left_motor  = Motor(Port.B)
right_motor = Motor(Port.C)
robot = DriveBase(left_motor, right_motor, wheel_diameter=55.5, axle_track=104)

bumper = TouchSensor(Port.S1)

ev3.screen.print("Натисніть, щоб почати")

# Чекайте, поки кнопка не буде натиснута.
while not bumper.pressed():
    wait(10)

ev3.speaker.beep()
robot.straight(300)   # їхати вперед 300 мм
robot.stop()
```

Ось другий приклад — передній бампер, який їхає вперед, поки не потиснеться щось:

```python
# Їхати вперед, поки датчик дотику не буде натиснут.
while True:
    if bumper.pressed():
        robot.stop()
        ev3.speaker.beep(880, 300)
        break
    else:
        robot.drive(150, 0)
    wait(10)

ev3.screen.print("Тиск!")
```

**Той же основний шаблон датчика:**
```
while True:
    read sensor
    if condition:
        react
    else:
        keep going
    wait(10)
```

## Частина 3 – Ваш Проект

Розробіть робота, який використовує **датчик дотику** та **два мотори** для досягнення мети, яку **ви виберете**.

### Правила

1. Ваш робот повинен використовувати **датчик дотику** для прийняття рішень
2. Ваш робот повинен використовувати **DriveBase** (два мотори) для руху
3. Ви повинні використовувати **`while` цикл** з читанням датчика всередині
4. Ви вирішуєте, яка **мета** робота — будьте творчими

### Деякі ідеї для Початку

- **Bumper bot** — їхати вперед, поки не потиснеться стіна, потім назад і повернути
- **Press-to-start** — сидіти нерухомо, і їхати лише після натиснення кнопки
- **Counting bot** — підраховує натиснення кнопок на екрані, поки ви не натиснете 10 разів, потім звучить гудок
- **Kill switch** — їхати по малюнку (коло, квадрат, цифра-8), поки не натиснеться кнопка, потім зупиниться
- **Two-press dance** — перше натиснення = обертання, друге натиснення = їхати вперед, третє натиснення = зупинка

### Ваш Контрольний Список Планування
