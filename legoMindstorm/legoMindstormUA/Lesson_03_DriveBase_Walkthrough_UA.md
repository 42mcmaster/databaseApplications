Перш ніж відкривати VS Code, прочитайте цю програму від початку до кінця. До кожного рядка є коментар. Ваше завдання зараз — **читати, а не вводити**.

```python
#!/usr/bin/env pybricks-micropython
# ↑ Цей рядок ОБОВ'ЯЗКОВО повинен бути першим рядком кожної програми для EV3.
#   Він повідомляє EV3 brick: "цей файл є програмою MicroPython."
#   Якщо видалити або перемістити його, програма не запуститься.

# -------------------------------------------------------
# IMPORTS — завантаження необхідних інструментів
# -------------------------------------------------------

from pybricks.hubs import EV3Brick
# ↑ "from X import Y" означає: перейди до бібліотеки pybricks, знайди розділ hubs,
#   і підключи інструмент EV3Brick.
#   EV3Brick дає нам контроль над екраном, динаміком та індикатором стану.

from pybricks.ev3devices import Motor
# ↑ З розділу ev3devices підключаємо Motor.
#   Це потрібно, тому що наші два колеса керуються двома об'єктами Motor.

from pybricks.parameters import Port
# ↑ З parameters підключаємо Port.
#   Port дозволяє нам вказати, *до якого роз'єму* підключений мотор: Port.B або Port.C.
#   Без цього ми не можемо сказати роботу, де шукати свої мотори.

from pybricks.robotics import DriveBase
# ↑ DriveBase — головний інструмент цього уроку.
#   Він приймає два мотори й об'єднує їх в один об'єкт "robot",
#   який може їхати прямо та повертати без того, щоб ми самі робили математику.

from pybricks.tools import wait
# ↑ wait() зупиняє програму на задану кількість мілісекунд.
#   1000 мс = 1 секунда. Ми використовуємо її для пауз між рухами.

# -------------------------------------------------------
# OBJECT CREATION — створення об'єктів для керування
# -------------------------------------------------------

ev3 = EV3Brick()
# ↑ Це створює об'єкт EV3Brick і зберігає його у змінній ev3.
#   Пам'ятайте з Уроку 01: *об'єкт* — це змінна, що представляє реальний
#   фізичний предмет. ev3.speaker, ev3.screen, ev3.light — все тепер працює.

left_motor = Motor(Port.B)
# ↑ Це створює об'єкт Motor для мотора, підключеного до порту B.
#   Ми зберігаємо його у змінній left_motor, щоб назва підказувала нам,
#   якою стороною робота він керує.
#   Port.B — це константа, яка означає "порт з позначкою B на brick."

right_motor = Motor(Port.C)
# ↑ Те ж саме, але для порту C, який керує правим колесом.
#   Наявність окремих змінних left_motor і right_motor робить
#   наступний рядок зрозумілішим.

robot = DriveBase(left_motor, right_motor, wheel_diameter=55.5, axle_track=104)
# ↑ Це найважливіший рядок налаштування.
#   DriveBase() приймає чотири аргументи:
#
#     left_motor     → об'єкт мотора для лівого колеса
#     right_motor    → об'єкт мотора для правого колеса
#     wheel_diameter → діаметр кожного колеса в міліметрах (55,5 мм)
#     axle_track     → відстань між двома колесами в міліметрах (104 мм)
#
#   Навіщо нам потрібні виміри?
#   Коли ми говоримо robot.straight(500), DriveBase потрібно розрахувати,
#   скільки разів має повернутися кожне колесо, щоб проїхати 500 мм.
#   Він може зробити цей розрахунок лише якщо знає розмір коліс.
#
#   Якщо wheel_diameter неправильний, робот проїде неправильну відстань.
#   Якщо axle_track неправильний, робот повернеться на неправильний кут.

# -------------------------------------------------------
# СТАРТОВИЙ СИГНАЛ
# -------------------------------------------------------

ev3.screen.clear()
# ↑ Очищає все, що залишилося на екрані з попереднього запуску.

ev3.screen.print("Ready!")
# ↑ Відображає "Ready!" на екрані EV3, щоб ми знали, що програма стартувала.

ev3.speaker.beep()
# ↑ Відтворює короткий сигнал. Це хороша звичка — підтверджує, що програма
#   запустилася успішно перед початком будь-яких рухів.
```

## Частина 2 – Рух робота

Тепер додаємо рух. Читайте цей розділ так само — до кожного рядка є коментар.

```python
# -------------------------------------------------------
# ЇЗДА ПРЯМО
# -------------------------------------------------------

robot.straight(400)
# ↑ Рухає робота ВПЕРЕД на 400 міліметрів (приблизно 16 дюймів), потім зупиняється.
#   Програма ЧЕ КАЄ тут — наступний рядок не виконується, поки цей рух не завершиться.
#   Це називається "блокуючою" командою.

wait(500)
# ↑ Після зупинки робота зробіть паузу на 500 мс (половину секунди) перед наступним рухом.
#   Це дає роботу момент для стабілізації, щоб рухи не зливались.

robot.straight(-400)
# ↑ Рухається НАЗАД на 400 мм. Знак мінус змінює напрямок на протилежний.
#   Правило Python: від'ємні числа означають протилежний напрямок у всьому pybricks.

wait(500)

# -------------------------------------------------------
# ПОВОРОТИ
# -------------------------------------------------------

robot.turn(90)
# ↑ Повертає робота ВПРАВО на 90 градусів на місці (чверть оберту).
#   Робот обертається навколо центральної точки між двома колесами —
#   одне колесо їде вперед, інше — назад.
#   Додатні градуси = поворот вправо (за годинниковою стрілкою).

wait(500)

robot.turn(-90)
# ↑ Повертає робота ВЛІВО на 90 градусів.
#   Від'ємні градуси = поворот вліво (проти годинникової стрілки).
#   Після цього робот повернувся до початкового напрямку.

wait(500)

robot.turn(360)
# ↑ Виконує повний оберт на 360 градусів на місці.
#   Після цього робот дивиться точно в тому ж напрямку, що й на початку.
```

**Правило знаку — запам'ятайте:**

| Додатне значення | Від'ємне значення |
|---|---|
| `robot.straight(400)` → вперед | `robot.straight(-400)` → назад |
| `robot.turn(90)` → вправо | `robot.turn(-90)` → вліво |

## Частина 3 – Використання циклу для повторення рухів

Цикл `for` дозволяє повторювати блок коду задану кількість разів. Це надзвичайно корисно, коли траєкторія має повторюваний шаблон.

Прочитайте програму квадрата нижче. Поки не вводьте.

```python
#!/usr/bin/env pybricks-micropython

from pybricks.hubs import EV3Brick
from pybricks.ev3devices import Motor
from pybricks.parameters import Port
from pybricks.robotics import DriveBase
from pybricks.tools import wait

ev3 = EV3Brick()
left_motor  = Motor(Port.B)
right_motor = Motor(Port.C)
robot = DriveBase(left_motor, right_motor, wheel_diameter=55.5, axle_track=104)

ev3.screen.clear()
ev3.screen.print("Square!")
ev3.speaker.beep()

# Квадрат має 4 сторони і 4 кути.
# Замість того щоб писати ті самі два рядки 4 рази, використовуємо цикл.

for i in range(4):
# ↑ "for i in range(4)" означає: повторити відступлений блок 4 рази.
#   i — це змінна, яка рахує ітерації: 0, 1, 2, 3.
#   range(4) генерує послідовність [0, 1, 2, 3] — це 4 значення, 4 ітерації.
#   Ми не використовуємо i для чогось конкретного — він просто рахує за нас.

    robot.straight(400)
    # ↑ Проїхати одну сторону квадрата (400 мм).
    #   Цей рядок ВІДСТУПЛЕНИЙ — це означає, що він всередині циклу.
    #   Він буде виконуватися один раз на ітерацію циклу.

    robot.turn(90)
    # ↑ Повернутися вправо на 90 градусів, щоб дивитися на наступну сторону.
    #   Також всередині циклу. Після 4 ітерацій:
    #     - 4 прямолінійних руху × 400 мм = 1600 мм загалом
    #     - 4 повороти × 90° = 360° загалом = повернулись до початку

ev3.speaker.beep(880, 500)
# ↑ НЕ відступлений — це поза циклом.
#   Виконується один раз після завершення всіх 4 ітерацій.

ev3.screen.print("Done!")
# ↑ Також поза циклом. Відображається після завершення квадрата.
```

**Правило відступу:**
Рядки, відступлені під `for i in range(4):`, знаходяться **всередині циклу** — вони повторюються.
Рядки на тому ж рівні, що й `for`, знаходяться **поза циклом** — вони виконуються один раз.

## Підсумок уроку

| Код | Що робить |
|---|---|
| `from pybricks.robotics import DriveBase` | Завантажує інструмент DriveBase |
| `left_motor = Motor(Port.B)` | Створює об'єкт мотора для порту B |
| `robot = DriveBase(left_motor, right_motor, wheel_diameter=55.5, axle_track=104)` | Створює трансмісію робота |
| `robot.straight(mm)` | Їде вперед (додатне) або назад (від'ємне) на задану відстань |
| `robot.turn(degrees)` | Повертає вправо (додатне) або вліво (від'ємне) на місці |
| `for i in range(n):` | Повторює відступлений блок n разів |
| `str(value)` | Перетворює число на рядок, щоб можна було вивести на екран |
| `ev3.screen.print("text")` | Відображає текст на екрані EV3 |
| `ev3.speaker.beep(freq, ms)` | Відтворює сигнал із заданою висотою тону та тривалістю |
| `wait(ms)` | Зупиняє програму (1000 мс = 1 секунда) |
